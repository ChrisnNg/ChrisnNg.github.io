Article Overview

You might have often heard that Javascript is a single-threaded language, but what does that really mean? A simple query will return:

Javascript has a single call stack and memory heap.

In layman's terms, it means that code is executed in the order it is read and won't move onto the next until it finishes. For those just beginning to learn to code, this might seem expected. Because who doesn't want their code to execute in order? But the words 'won't move onto the next until it finishes' can carry a nefarious implication.
